# Create Chat Completion | DeepSeek API Docs

URL: https://api-docs.deepseek.com/api/create-chat-completion

## Request

### messages

A list of messages comprising the conversation so far.

**Assistant message fields:**
- `content`: string | null — The contents of the assistant message.
- `role`: string — `"assistant"`
- `name`: string — optional
- `prefix`: bool — (Beta) Chat Prefix Completion
- `reasoning_content`: string | null — (Beta) Used for the thinking mode in the Chat Prefix Completion feature as the input for the CoT in the last assistant message. When using this feature, the `prefix` parameter must be set to `true`.

### thinking

Controls the switch between thinking and non-thinking mode.

- `type`: string — `"enabled"` | `"disabled"`, default `"enabled"`
- `reasoning_effort`: string — `"high"` | `"max"`, default `"high"` for regular requests; for some complex agent requests effort is automatically set to `"max"`. For compatibility, `low` and `medium` are mapped to `high`, and `xhigh` is mapped to `max`.

### tools

A list of tools the model may call. Currently, only functions are supported.

- `type`: `"function"`
- `function`: object with `description`, `name`, `parameters`
- `strict`: boolean, default `false`

### tool_choice

Controls which (if any) tool is called by the model.

Possible values: `"none"`, `"auto"`, `"required"`, or specific tool via `{"type": "function", "function": {"name": "..."}}`

## Response (Non-Streaming)

Schema:
```json
{
  "id": "string",
  "choices": [
    {
      "finish_reason": "stop",
      "index": 0,
      "message": {
        "content": "string",
        "reasoning_content": "string",
        "tool_calls": [
          {
            "id": "string",
            "type": "function",
            "function": {
              "name": "string",
              "arguments": "string"
            }
          }
        ],
        "role": "assistant"
      },
      "logprobs": {
        "content": [...],
        "reasoning_content": [...]
      }
    }
  ],
  "created": 0,
  "model": "string",
  "system_fingerprint": "string",
  "object": "chat.completion",
  "usage": {
    "completion_tokens": 0,
    "prompt_tokens": 0,
    "prompt_cache_hit_tokens": 0,
    "prompt_cache_miss_tokens": 0,
    "total_tokens": 0,
    "completion_tokens_details": {
      "reasoning_tokens": 0
    }
  }
}
```

**Key fields on `message`:**
- `content`: string | null — The contents of the message
- `reasoning_content`: string | null — For thinking mode only. The reasoning contents of the assistant message, before the final answer.
- `tool_calls`: array — The tool calls generated by the model
- `role`: `"assistant"`

**Logprobs:** `logprobs.reasoning_content` is a separate array of tokens for reasoning content.

## Response (Streaming)

Content-Type: `text/event-stream`

**Delta object fields:**
- `content`: string | null — The contents of the chunk message
- `reasoning_content`: string | null — For thinking mode only. The reasoning contents of the assistant message, before the final answer.
- `role`: `"assistant"`

Example SSE chunks:
```
data: {"id": "...", "choices": [{"index": 0, "delta": {"content": "", "role": "assistant"}, ...}], ...}
data: {"choices": [{"delta": {"content": "Hello", ...}, ...}]}
...
data: [DONE]
```

## Models

Supported models: `deepseek-v4-flash`, `deepseek-v4-pro`

Both models support `thinking` mode.
